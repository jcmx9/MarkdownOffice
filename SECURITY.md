# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Older releases | No |

Only the latest release receives security updates.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities via public issues.

Instead, use one of:

1. **GitHub Security Advisories** — open a private report via the repository's
   *Security → Report a vulnerability* tab (preferred).
2. **Email** — send a description to **jcmx9@tutamail.com**.

Please include steps to reproduce if possible. We will acknowledge receipt within
**48 hours** and aim to provide a fix within **14 days**.

After a fix is released, the vulnerability will be documented in the CHANGELOG.

## Principles

- No retaliation against vulnerability reporters.
- Coordinated disclosure: publication only after a fix is available.
- Credit in the CHANGELOG and release notes (if desired).
